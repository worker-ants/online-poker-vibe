# AI 플레이어 스펙

## 개요
방에 접속된 모든 사용자가 준비완료 상태일 때, 설정된 최대 플레이어 수(`maxPlayers`)에서 부족한 플레이어를 AI 플레이어로 채우고 즉시 게임을 시작한다.

## AI 플레이어 정의
- 서버사이드에서만 동작하는 코드 기반 플레이어 (LLM 미사용)
- 다른 플레이어의 홀카드를 열람할 수 없음 (자신의 카드 + 커뮤니티/비지블 카드만 참조)
- 랭킹 시스템(Hall of Fame)에 반영되지 않음
- 빠른 게임 진행을 위해 delay 없이 즉시 액션 수행

## 식별 방식
- UUID: `ai-player-{n}` 형식 (예: `ai-player-1`, `ai-player-2`)
- 닉네임: `[AI] {이름}` 형식 (예: `[AI] Alice`, `[AI] Bob`)
- 이름 풀: Alice, Bob, Charlie, Diana, Eve, Frank
- Player DB 테이블에 저장하지 않음 (인메모리 전용)
- GameParticipant 레코드를 생성하지 않음 → 랭킹 자동 제외

## 게임 시작 조건 변경
- 기존: 모든 플레이어 준비완료 + 최소 2명
- 변경: 모든 플레이어 준비완료 + 최소 1명 (AI로 나머지 채움)
- AI 플레이어 수 = `maxPlayers - 현재 인간 플레이어 수`

## AI 의사결정 전략

### 핸드 강도 평가
1. **프리플롭 (Texas Hold'em)**: 룩업 테이블 사용
   - 포켓 페어: AA=1.0, KK=0.95, QQ=0.9, ... 22=0.5
   - 수트 커넥터: AKs=0.85, KQs=0.75 등
   - 오프수트: AKo=0.75, KQo=0.65 등
2. **포스트플롭 / 일반**: HandEvaluator 사용
   - HandCategory별 점수: HighCard=0.1 ~ RoyalFlush=1.0

### 의사결정 매트릭스
| 핸드 점수 | 액션 |
|-----------|------|
| >= 0.8 | 공격적 레이즈 (팟 크기 또는 3x BB) |
| >= 0.6 | 소폭 레이즈 또는 콜 |
| >= 0.4 | 팟 오즈가 유리하면 콜, 아니면 체크/폴드 |
| >= 0.2 | 무료면 체크, 베팅 있으면 폴드 (블러프 10%) |
| < 0.2 | 폴드 (블러프 5%) |

### 팟 오즈 계산
```
effectiveOdds = callAmount / (pot + callAmount)
handScore > effectiveOdds × 1.5 → 콜/레이즈 유리
```

### Five Card Draw 드로우 로직
- 페어/트립스/쿼드 유지
- 플러시/스트레이트 4장 유지
- 그 외: 가장 높은 카드 유지, 나머지 버림

### 블러프
- 약한 핸드(0.2~0.4)에서 10% 확률로 블러프 레이즈
- 매우 약한 핸드(<0.2)에서 5% 확률로 블러프 레이즈

## 게임 흐름 통합
1. 모든 인간 플레이어 준비완료
2. AI 플레이어 생성 및 빈 좌석 배치
3. `gameService.startGame()` 호출 시 AI 포함
4. 각 액션 후 다음 턴 플레이어가 AI이면 즉시 `processAiTurns()` 실행
5. AI 턴 루프: AI 액션 → 상태 브로드캐스트 → 다음 플레이어 확인 → 반복
6. 인간 플레이어 턴이 되면 `GAME_ACTION_REQUIRED` 이벤트 전송

## 프론트엔드 표시
- `PlayerPublicState`에 `isAI` 플래그 포함
- AI 플레이어는 보라색 `AI` 뱃지로 구분
- 대기실(Waiting)에서는 AI 미표시 (게임 시작 시에만 생성)

## 랭킹 시스템 처리
- AI 플레이어의 GameParticipant 레코드를 생성하지 않음
- 방어적 필터: 랭킹 쿼리에 `playerUuid NOT LIKE 'ai-%'` 조건 추가
